Термин/Объяснение

Полиморфизм - пособность объектов разных классов иметь один интерфейс, но разную реализацию (через виртуальные методы)

Инкапсуляция - данные (data_, rows_, cols_) — приватные, доступ через методы Get()/Set()

Конструктор по умолчанию - Matrix() — создает пустую матрицу (нулевого размера)

Конструктор копирования	- Matrix(const Matrix& other) — создает глубокую копию

Деструктор	~Matrix() - освобождает динамическую память (delete[])

Глубокая копия - Копируется не только указатель, но и сами данные (новые массивы в куче)

Оператор присваивания	operator= - освобождает старые данные, копирует новые, возвращает *this

Правило трех	- если нужен деструктор → нужны конструктор копирования и оператор присваивания

Бинарный оператор	+, -, *, / - работают с двумя операндами

Унарный оператор	++, --, - (унарный минус) — работают с одним операндом

Префиксный инкремент	++m - возвращает ссылку на измененный объект (Matrix&)

```
BigInt& BigInt::operator++() {

    *this += BigInt(1);  // добавляем 1
    return *this;        // возвращаем ссылку на себя
}
```
Постфиксный инкремент	m++ - возвращает копию старого значения, принимает фиктивный int

```
BigInt BigInt::operator++(int) {

    BigInt temp = *this;  // запоминаем старое значение
    ++(*this);            // увеличиваем текущий объект (префиксным)
    return temp;          // возвращаем старое значение
}
```
Константность и операторы	operator+ const - не изменяет текущий объект

Дружественная функция	friend - имеет доступ к приватным членам (для operator<< в lab5)

Куча (heap)	- область памяти для динамического выделения (new/delete)

Утечка памяти	- потеря указателя на выделенную память без вызова delete

Абстрактный класс	- имеет хотя бы один чисто виртуальный метод (= 0)

Чисто виртуальный метод	- метод без реализации в базовом классе

TCP	- протокол с установкой соединения, гарантирует доставку

IP-адрес - адрес устройства в сети (например, 127.0.0.1)

Порт	- номер, по которому программа слушает соединения (0–65535)

Сокет	 - программный интерфейс для сетевого обмена (socket - отправляет и получает данные)

Endpoint - пара (IP-адрес + порт)

ASIO	- библиотека для асинхронного ввода-вывода (сеть, таймеры)

Сериализация - превращение структуры данных в байты для передачи по сети

Клиент-серверная архитектура - одна программа (сервер) предоставляет данные, другая (клиент) их запрашивает

Инкапсуляция - данные скрыты внутри класса (params_ приватный)

Бинарный протокол - данные передаются в виде байтов, а не текста

Выравнивание (padding) - компилятор добавляет пустые байты между полями структуры

Порядок байт (endianness) - Little-endian vs Big-endian (как записаны многобайтовые числа)

Периодический опрос - автоматическое повторяющееся получение данных со всех метеостанций

Конфигурирование - хранение настроек (списка станций) в отдельном файле

Синхронный режим - программа ждёт завершения операции, прежде чем двигаться дальше

Сервер - программа, которая ждёт подключений от других программ (клиентов) и отвечает на их запросы (однопоточный)

Субмодули - отдельные составные части программы, каждая из которых отвечает за свою задачу. Вместе они образуют целую систему

Нетривиальный деструктор	- деструктор, который освобождает ресурсы (память, файлы, сокеты)

Тривиальный класс	- нет динамической памяти, компилятор сам генерирует конструкторы/деструктор

Copy-and-swap	- техника безопасного оператора присваивания;
Алгоритм	1. Создать временную копию 2. Обменять данные с временной 3. Автоматическое удаление временной

```
Matrix& operator=(const Matrix& other) {

    Matrix tmp = other;           // 1. копия
    std::swap(tmp.rows_, rows_);  // 2. обмен
    std::swap(tmp.cols_, cols_);
    std::swap(tmp.data_, data_);
    return *this;                 // 3. tmp удаляется сам
}
```

ВОЗВЕДЕНИЕ В СТЕПЕНЬ pow()

```
BigInt BigInt::pow(int exponent) const {

    if (exponent < 0) {
        throw std::invalid_argument("Отрицательный показатель степени не поддерживается");
    }
    BigInt result(1);      // начинаем с 1
    BigInt base = *this;   // копируем основание
    while (exponent > 0) {
        if (exponent % 2 == 1) {   // если текущий бит степени = 1
            result *= base;         // умножаем результат на base
        }
        base *= base;              // возводим base в квадрат
        exponent /= 2;             // делим степень на 2
    }
    return result;
}
```
```
Начало: result = 1
        base = a
        exponent = n
Пока exponent > 0:
    Если exponent нечетный (exponent % 2 == 1):
        result = result × base
    base = base × base
    exponent = exponent ÷ 2 (целочисленное деление)
Конец цикла
Ответ: result = aⁿ
```

```
BigInt BigInt::sqrt() const {

    if (isNegative) {
        throw std::invalid_argument("Квадратный корень из отрицательного числа");
    }
    if (isZero()) {
        return BigInt(0);
    }
    BigInt left(1);
    BigInt right = *this;
    BigInt result(0);
    while (left <= right) {
        BigInt mid = (left + right) / BigInt(2);  // бинарный поиск
        BigInt square = mid * mid;
        if (square == *this) {
            return mid;           // точный корень
        } else if (square < *this) {
            result = mid;         // запоминаем кандидата
            left = mid + BigInt(1);
        } else {
            right = mid - BigInt(1);
        }
    }
    return result;  // целая часть корня (округление вниз)
}
```

для 13 пример 
left=1, right=13
mid=7 → 7²=49 > 13 → right=6
mid=3 → 3²=9 < 13 → result=3, left=4
mid=5 → 5²=25 > 13 → right=4
mid=4 → 4²=16 > 13 → right=3
left=4, right=3 → выход, result=3

КОРЕНЬ N-Й СТЕПЕНИ nRoot()

```
BigInt BigInt::nRoot(int n) const {

    if (n <= 0) {
        throw std::invalid_argument("Степень корня должна быть положительной");
    }
    if (isNegative) {
        if (n % 2 == 0) {
            throw std::invalid_argument("Чётный корень из отрицательного числа");
        }
        BigInt absResult = (-(*this)).nRoot(n);  // корень из модуля
        return -absResult;                       // со знаком минус
    }
    if (isZero()) return BigInt(0);
    if (n == 1) return *this;
    if (n == 2) return this->sqrt();  // используем готовый метод
    BigInt left(1);
    BigInt right = *this;
    BigInt result(0);
    while (left <= right) {
        BigInt mid = (left + right) / BigInt(2);
        BigInt power = mid.pow(n);   // возводим mid в степень n
        if (power == *this) {
            return mid;
        } else if (power < *this) {
            result = mid;
            left = mid + BigInt(1);
        } else {
            right = mid - BigInt(1);
        }
    }
    return result;  // целая часть корня}
}
```

Алгоритм:
Начало: left = 1, right = A
Середина: mid = (left + right) / 2
Вычисляем midⁿ (возводим в степень n)
Сравниваем с A:
Если midⁿ = A → нашли точный корень
Если midⁿ < A → корень больше, ищем справа (left = mid + 1)
Если midⁿ > A → корень меньше, ищем слева (right = mid - 1)
Повторяем, пока left ≤ right

Страшные вопросы

Почему цифры в BigInt хранятся в обратном порядке?	Чтобы при сложении/вычитании перенос шел естественно от младших разрядов к старшим

Чем отличается ++m от m++? Префиксный возвращает ссылку на измененный объект, постфиксный — копию старого значения

Что такое чисто виртуальный метод?	Метод без реализации в базовом классе, делает класс абстрактным
